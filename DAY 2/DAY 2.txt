리액트는 이전에 렌더링된 컴포넌트는 어떤 거였는지 확인하고 있다.
그리고 다음에 렌더링 될 컴포넌트는 어떤지를 보고,
리액트는 다른 부분만 파악한다. => 우리가 바꾼 부분
바뀐 부분만 업데이트 해주면 된다. => 굉장히 인터랙티브 한 어플을 만들 수 있다는 뜻

여러가지 요소들을 리렌더링 하려고 해도 전부 다 새로 생성되지는 않는다.
오로지 바뀐 부분만 생성된다.

---- 
React.js 어플 내에서 데이터를 보관하고 자동으로 리렌더링을 일으킬 수 있는 최고의 방법



React.useState();
line Babel script:7 (2) [undefined, ƒ]
배열의 첫번째 값은 초기값
그 두번째는 값을 바꾸는 함수

React.useState();은 초기값을 설정할 수 있다.
React.useState(0);
=> (2) [0, ƒ ] counter 라는 data를 주고, 그 counter 값을 바꿀 수 있는 함수를 주고있다.
 

[recap]
React 컴포넌트 안에서 데이터를 어떻게 바꾸는지 배워봤고,
우리가 데이터를 룰에 맞춰 변경했을 때, React가 화면을 어떻게 refresh 하는지 알아봤다
방금 말한 refresh가 바로 리렌더링이다.
함수를 사용해 컴포넌트의 state를 바꿀 때 컴포넌트는 새로운 값을 가지고 다시 한번 리렌더링된다. => 컴포넌트가 재생성된다.
